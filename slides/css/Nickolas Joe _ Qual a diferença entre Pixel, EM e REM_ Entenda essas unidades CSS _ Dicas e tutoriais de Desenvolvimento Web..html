Qual a diferença entre Pixel, EM e REM? Entenda essas unidades CSS

Publicado em 26 de janeiro de 2021

Você sabe a diferença entre usar px, em ou rem no CSS? Já se perguntou qual dessas unidades é melhor? Esta é uma dúvida que já recebi bastante no meu canal do YouTube, e hoje vamos aprender a diferença entre cada uma delas.

Pixel

Pixel, ou px, é um valor fixo.

Ele serve muito bem para telas, e é o valor mais comumente usado.

p {
  font-size: 14px;
}

No final, tudo vira pixel no site. Mas e se o usuário decidisse ver esses pixels um pouco maiores em seu dispositivo? Se você definiu eles com um valor fixo sempre, eles não irão aumentar de tamanho, e isso pode ser considerado um problema de acessibilidade. Vamos tentar então encontrar uma solução nas unidades a seguir.

EM

Quando a unidade em surgiu, foi uma verdadeira revolução.

Agora tamanhos de tipografia passam a usar uma unidade relativa ao tamanho de fonte que o usuário escolher. Se o usuário decidir deixar as letras bem grandes no navegador ou dispositivo móvel dele (devido a ele enxergar pouco ou qualquer outro motivo) e o desenvolvedor utilizou a unidade em no site, as letras irão se adaptar às suas preferências. Geralmente, o tamanho padrão para 1em é de 16px. 2em seria 32px (pois 16 x 2 = 32), e assim por diante.

p {
  font-size: 1em;
}

O nome em significa emphemeral unit, ou traduzindo para o português, unidade transitória! 🙂

Qual talvez poderia ser um problema ao usar esta unidade?

Faça o teste: se você definir font-size: 14px no <html> do seu site, e colocar font-size: 2em em uma <section>, ele será de 28px, pois 14 x 2 = 28. Até aí, nada de mais. O problema é que se você definir font-size: 2em novamente, digamos, para uma <ul> dentro desta <section>, ele será de 56px, que é o dobro do dobro. Logo, isso pode acabar se tornando inconsistente e bastante complexo em um site grande, pois em sempre é uma unidade relativa ao elemento pai.

A solução para isso se encontra na unidade a seguir.

REM

Finalmente, chegamos ao futuro (ou deveria dizer, presente) das unidades CSS.

Esperava-se esta unidade desde meados de 2010, conforme comentado por ninguém menos do que um dos criadores do CSS, Bert Bos, neste artigo da W3C, mas ela só foi lançada em 2013.

O nome rem significa Root EM. Se você definir na raiz do seu site font-size: 14px, 1rem sempre será 14px em absolutamente qualquer elemento, corrigindo assim o problema do em com elementos filhos.

html {
  font-size: 14px;
}
/*
  Definindo um font-size na raiz. Obs.: Por padrão o valor é 16px, mas você pode colocar qualquer valor.
*/

section {
  font-size: 2rem;
}
/*
  A section terá um font-size de 28px, pois (14 x 2 = 28)
*/

ul {
  font-size: 2rem;
}
/*
  Usando rem ao invés de em, o font-size da ul que está dentro da section também será 28px, e não 56px! 😉
*/
